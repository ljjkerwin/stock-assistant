# 暗盘资金模块（DarkTrade）

> 本文档是 [AGENTS.md](../../AGENTS.md) 的卫星文档。改动暗盘抓取、索引或字段映射时**同步更新本文件**。接口行见 [docs/api.md 暗盘资金](../api.md#暗盘资金)。

---

## 工作机制

- **定时自动重建**：每天北京时间 **09:28（仅限开盘交易日，周一至周五）**，系统中央心跳服务会自动触发全量索引重建，确保在开盘（09:30）前索引已就绪。
- K线总览页在加载时会自动携带客户端今日日期调用 `GET /api/darktrade/snapshots-batch?codes=&date=YYYYMMDD`，后端检测到索引日期不匹配时自动作为兜底在服务端并发锁中触发全量索引刷新（爬取约 177 页 ~5300 只股票，约 5–10 秒），**无需手动运行脚本**
- 手动刷新仍可用 `node scripts/refresh-darktrade-index.mjs [YYYYMMDD]`（调用 `POST /api/darktrade/refresh-index`）
- `refresh-index` 建立 `code → (page, index)` 映射；`snapshots-batch` / `:code` 通过该映射定位再取数；`date` 不匹配时，`snapshots-batch` 接口同样会在服务端用并发锁触发一次刷新（多请求只刷新一次）
- **静态排序防飘移**：建立索引和抓数时统一默认采用静态的股票名称排序（`sortFlag=4`）。因为股票名称的 Unicode 顺序是绝对静态不变的，这规避了按暗盘资金等动态指标排序（`sortFlag=6`）时，因股票排名盘中剧烈变化而导致其漂移到其他页面，进而导致接口请求错失并回退至静态旧数据（使明暗盘数据看似卡死、不更新）的 Bug。
- **自愈重构索引**：即便采用名称排序，若盘中发生交易的股票总数增加（新股票获得交易被计入），股票的相对页码仍可能发生微调（例如 300059 东方财富在 09:30 位于第 150 页，盘中随前序字母股票增多而漂移到第 153 页）。为此，服务层实现了自愈机制：在获取最新数据时，若发现已索引的标的未在其对应页码中找到，且距离上次刷新已超过 5 分钟，将自动触发索引重构并二次拉取，确保分时数据能够实时更新。

## 暗盘快照（分时副图）

- 后端按交易时段定时抓取快照写入 `dark_trade_snapshots`（开盘前 09:30 之前的快照过滤不写入/不返回），形成**当日分钟粒度**的暗盘/明盘资金累计序列
- **收盘补点**：09:30–15:00 期间按抓取时的实际分钟打 `captureMinute`（最迟落在 14:5x，到不了 15:00）；**收盘后（北京时间 ≥15:00）再次拉取时，源数据已是当日收盘终值，会补写为 `${date}1500` 的 15:00 收盘快照**，填上 242 槽位横轴最后一槽。即只要收盘后再加载一次总览/详情页，当日就有完整到 15:00 的序列
- **股票详情页分时图**与 **K 线总览页**的暗盘均通过 `GET /api/darktrade/snapshots-batch?codes=&date=YYYYMMDD` 按**当日交易日**拉取分钟粒度快照。为了节省网络请求开销，K 线总览页去掉了对 `/api/darktrade/batch` 的依赖，卡片右上角的明暗盘最新汇总值也是直接从该快照列表的最后一根中提取数据进行渲染。其中详情页的交易日由 `KLineChart` 加载 K 线后经 `onDateResolved` 上报「K 线实际交易日」（而非客户端日历日，规避周末/盘前不一致），父组件据此调 snapshots-batch 取 `Record[code]`
- 前端 `KLineChart` 与总览页暗盘副图：明盘蓝线 + 暗盘黄线 + 总中灰折线，横轴用 241 槽位（09:30–11:30 + 13:01–15:00，午休不画）与分时主图逐槽对齐，无快照的分钟留空
- `GET /api/darktrade/snapshots/:code?days=` 为**日粒度**历史快照（每日取最后一条），当前前端未使用，保留备用

## 数据源与编码

- 数据源：东方财富 `quotederivates.eastmoney.com/datacenter/darktrade`，支持 `date=YYYYMMDD` 参数
- 返回 **GBK 编码**，服务层用 `TextDecoder('gbk')` 解码，**无需额外安装 iconv-lite**

## 字段映射

| 键 | 含义 |
|----|------|
| `"6"` | 暗盘资金（元） |
| `"7"` | 明盘资金（元） |
| `"8"` | 主力净流入含暗盘（元） |
| `"11"` | 暗盘活跃度（小数） |
| `"13"` | 最新价 × 1000 |
| `"14"` | 涨幅（小数） |
| `"16"` | 名称 |
| `"17"` | 行业 |
| `"18"` | 概念 |
